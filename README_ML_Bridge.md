# ML Bridge для MetaTrader 5

## Описание

ML Bridge - это мост между Python и MetaTrader 5 для автоматической генерации торговых сигналов на основе технических индикаторов.

## Основные возможности

- ✅ Надежное чтение данных из CSV файлов MT5
- ✅ Улучшенная генерация сигналов с учетом ATR
- ✅ Подробное логирование всех операций
- ✅ Сбор статистики работы
- ✅ Мониторинг в реальном времени
- ✅ Обработка ошибок и восстановление
- ✅ Атомарная запись файлов

## Файлы проекта

- `ml_bridge.py` - Основной модуль ML Bridge
- `ml_bridge_monitor.py` - Утилита мониторинга
- `ml_bridge_config.json` - Конфигурационный файл
- `README_ML_Bridge.md` - Документация

## Установка и настройка

### 1. Требования

- Python 3.7+
- MetaTrader 5
- Библиотеки: pandas, numpy, scipy

### 2. Настройка путей

Отредактируйте пути в `ml_bridge_config.json`:

```json
{
  "paths": {
    "features_file": "C:\\Users\\YourUser\\AppData\\Roaming\\MetaQuotes\\Terminal\\Common\\Files\\features_bt.csv",
    "prediction_file": "C:\\Users\\YourUser\\AppData\\Roaming\\MetaQuotes\\Terminal\\Common\\Files\\prediction_bt.txt"
  }
}
```

### 3. Настройка MT5

В MetaTrader 5 создайте эксперт, который:
- Записывает данные в формате: `YYYY.MM.DD HH:MM;close;ema;atr`
- Читает сигналы из файла `prediction_bt.txt`

## Использование

### Запуск ML Bridge

```bash
python ml_bridge.py
```

### Мониторинг

```bash
python ml_bridge_monitor.py
```

## Алгоритм генерации сигналов

### Простая логика (без ATR)
- BUY: `close > ema * 1.001`
- SELL: `close < ema * 0.999`
- NONE: в остальных случаях

### Улучшенная логика (с ATR)
- Динамические пороги на основе волатильности
- BUY: `close > ema * (1.0 + atr_threshold * 0.5)`
- SELL: `close < ema * (1.0 - atr_threshold * 0.5)`

## Логирование

Все операции записываются в файл `ml_bridge.log`:
- Генерация сигналов
- Ошибки чтения/записи
- Статистика работы

## Статистика

Статистика сохраняется в `ml_bridge_stats.json`:
- Количество сигналов по типам
- Количество ошибок
- Время последнего сигнала
- Время работы

## Мониторинг

Утилита `ml_bridge_monitor.py` предоставляет:
- Текущий статус работы
- Последние сигналы
- Статистику в реальном времени
- Просмотр логов

## Конфигурация

Основные параметры в `ml_bridge_config.json`:

```json
{
  "settings": {
    "poll_interval": 0.1,        // Интервал опроса (сек)
    "write_retries": 5,          // Попытки записи
    "cooldown_seconds": 0.5      // Пауза после записи
  },
  "signal_thresholds": {
    "buy_threshold": 1.001,       // Порог для BUY
    "sell_threshold": 0.999,      // Порог для SELL
    "min_price_difference": 0.01 // Минимальная разница
  }
}
```

## Устранение неполадок

### Проблемы с файлами
- Проверьте права доступа к папке Common/Files
- Убедитесь, что MT5 не блокирует файлы
- Проверьте кодировку файлов (UTF-8)

### Проблемы с сигналами
- Проверьте формат входных данных
- Убедитесь в корректности значений EMA и ATR
- Проверьте логи на наличие ошибок парсинга

### Производительность
- Увеличьте `poll_interval` для снижения нагрузки
- Уменьшите `write_retries` для быстрого восстановления
- Мониторьте статистику ошибок

## Безопасность

- Все файлы создаются с временными именами
- Атомарная запись предотвращает повреждение данных
- Автоматическая очистка временных файлов
- Подробное логирование для аудита

## Поддержка

При возникновении проблем:
1. Проверьте логи в `ml_bridge.log`
2. Запустите мониторинг для диагностики
3. Проверьте статистику ошибок
4. Убедитесь в корректности конфигурации
