# 🚀 Быстрая шпаргалка по Forecast Metrics Template

## ⚡ За 60 секунд

1. **Откройте** `Forecast_Metrics_Template.xlsx`
2. **Лист "Ввод_данных"** → заполните зелёные ячейки (C, D)
3. **Лист "Метрики"** → смотрите результаты
4. **Готово!** 

## 📊 Какую метрику использовать?

| Ситуация | Метрика | Почему |
|----------|---------|--------|
| Отчёт для руководства | **WAPE** | Учитывает объёмы, понятна |
| Сравнение моделей | **MAPE** | Стандартная метрика (если нет нулей) |
| Простая оценка | **MAE** | В штуках, интуитивно понятна |
| Поиск выбросов | **RMSE** | Усиливает большие ошибки |
| Проверка смещения | **Bias** | Показывает направление ошибки |

## 🎯 Целевые значения метрик

| Метрика | Отлично | Хорошо | Приемлемо | Плохо |
|---------|---------|--------|-----------|-------|
| **WAPE** | <5% | 5-10% | 10-15% | >15% |
| **MAPE** | <10% | 10-15% | 15-20% | >20% |
| **Bias/MAE** | <0.2 | 0.2-0.5 | 0.5-1.0 | >1.0 |

*Примечание: Bias/MAE - относительное смещение*

## ⚠️ Частые ошибки и решения

### Ошибка 1: #Н/Д в MAPE
**Причина:** Есть нулевые фактические значения  
**Решение:** Используйте WAPE вместо MAPE

### Ошибка 2: RMSE >> MAE
**Причина:** Есть выбросы (акции, всплески спроса)  
**Решение:** 
- Найдите оранжевые ячейки на листе "Ввод_данных"
- Проанализируйте причины выбросов
- Создайте отдельные модели для акций

### Ошибка 3: Bias ≈ 0, но WAPE высокий
**Причина:** Ошибки компенсируют друг друга (±)  
**Решение:** 
- Смотрите на MAE и WAPE
- Анализируйте по сегментам (сезоны, SKU)

## 🔍 Диагностическая таблица

| Признак | Диагноз | Действие |
|---------|---------|----------|
| WAPE > 15% | Низкая точность | Проверить качество данных, добавить факторы |
| Bias > 0 | Недопрогноз | Увеличить параметры модели |
| Bias < 0 | Перепрогноз | Уменьшить параметры модели |
| RMSE/MAE > 1.5 | Выбросы | Сгладить или выделить акции |
| Жёлтые ячейки | Нули в данных | MAPE недоступен, используйте WAPE |
| Оранжевые ячейки | Топ-10% ошибок | Исследовать причины |

## 📋 Чек-лист проверки прогноза

### Шаг 1: Ввод данных ✓
- [ ] Заполнены SKU, Период, Факт, Прогноз
- [ ] Нет пустых пар (если есть факт, должен быть прогноз)
- [ ] Все значения ≥ 0

### Шаг 2: Проверка качества ✓
- [ ] WAPE < 15% (приемлемо для большинства случаев)
- [ ] Нет массовых жёлтых ячеек (доля нулей < 30%)
- [ ] Оранжевые ячейки объяснимы (акции, события)

### Шаг 3: Анализ смещения ✓
- [ ] |Bias| < 0.5 × MAE (незначительное смещение)
- [ ] Если Bias > 0 → проверить склонность к недопрогнозу
- [ ] Если Bias < 0 → проверить склонность к перепрогнозу

### Шаг 4: Диагностика выбросов ✓
- [ ] RMSE / MAE < 1.5 (выбросов мало)
- [ ] Если RMSE >> MAE → найти причины выбросов

### Шаг 5: Визуальная проверка ✓
- [ ] График "Факт vs Прогноз" - линии близки
- [ ] График ошибок - нет систематических паттернов

## 🔧 Быстрые исправления

### Проблема: Высокий WAPE
```
1. Проверить сезонность → добавить сезонные коэффициенты
2. Проверить акции → учесть промо-периоды
3. Проверить тренд → добавить трендовую компоненту
4. Проверить данные → удалить аномалии
```

### Проблема: Положительный Bias (недопрогноз)
```
1. Увеличить базовый прогноз на величину Bias
2. Проверить упущенные факторы (новые каналы, маркетинг)
3. Пересмотреть параметры модели
```

### Проблема: Отрицательный Bias (перепрогноз)
```
1. Уменьшить базовый прогноз на величину |Bias|
2. Проверить устаревшие предпосылки
3. Учесть каннибализацию или снижение спроса
```

### Проблема: Много выбросов (RMSE >> MAE)
```
1. Найти оранжевые ячейки
2. Выделить особые периоды (акции, праздники)
3. Создать отдельные модели для этих периодов
4. Или: применить робастные методы (медиана вместо среднего)
```

## 📱 Мобильная версия формул

### Для копирования в Excel/Sheets:

**WAPE (основная метрика):**
```
=SUM(F:F)/SUM(C:C)
```

**MAE (средняя ошибка):**
```
=AVERAGE(F:F)
```

**Bias (смещение):**
```
=AVERAGE(E:E)
```

**Есть ли выбросы:**
```
=SQRT(SUMSQ(E:E)/COUNT(E:E))/AVERAGE(F:F)
Если > 1.5 → есть выбросы
```

### Для аналитиков:
1. **WAPE** - главная метрика для отчётов

### Для руководителей:
1. Смотрите только на **WAPE** и **Bias**
2. WAPE < 10% → можно доверять прогнозу
3. Bias показывает, куда мы ошибаемся (+ или -)

## 📞 Контрольные вопросы

**Q: Что важнее - MAPE или WAPE?**  
A: WAPE для портфелей и отчётов, MAPE для сравнения моделей (если нет нулей)

**Q: Какой WAPE считается хорошим?**  
A: <10% - хорошо, <5% - отлично, >15% - плохо

**Q: Что делать, если Bias = 0, но прогноз плохой?**  
A: Смотрите MAE и WAPE. Bias может маскировать проблему (+ и - компенсируются)

**Q: Как часто пересматривать метрики?**  
A: Еженедельно - оперативный контроль, ежемесячно - глубокий анализ

**Q: Нужно ли стремиться к Bias = 0?**  
A: Нет! Для разных бизнесов оптимален разный Bias (лучше недопрогноз или перепрогноз)
